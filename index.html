    //In this lab, you'll be creating Epics, writing User Stories, creating a Product Backlog, and prioritizing work items.

    Description: An up-and-coming start-up led by two healthcare prodigies, Riley and Kevin, has secured funding to develop Brosnan-Jeter Prestige Healthcare Worldwide.     Outlined below are some of their most critical business goals, in order of importance:

  1. Based on the goals, stakeholder needs, and requested features, identify 3 Epics that represent the major feature groups or outcomes for the MVP.

    Creating Epics:

    1. An Epic should be broad enough to include 3–7 user stories, but not so vague it loses focus.
    2. If you find a cluster of related user stories, group them under a single Epic. Think of an Epic as a container that makes backlog management more manageable and thematic.
      If several stories start with “As a patient, I want to…” related to communication, they likely belong to an Epic about messaging or interaction.
    3. It's ok to start with User Stories first and then come back to catergorize you User Stories into Epics. Agile isn't prescriptive - it's what works best for YOU.


    Epic Title Description Primary Stakeholders Impacted
    Virtual Consultations Enable secure, HIPAA-compliant real-time video calls between patients and PCPs Riley Brosnan, Dr. Patel, Daniel Kwon

  2. For the Epics you created, write 4 User Stories. Ensure your stories are meeting the INVEST criteria!

    Creating User Stories:

    "As a [user type], I want to [action or goal] so that [benefit or reason]"

     Hints for writing:
      1. Avoid generic phrases like "As a user." Be specific as to which user (e.g. patient, provider, admin, etc.)
      2. Include the Why! Instead of saying "so that I can use the app," say "so that I can follow up on my treatment plan"
      3. Keep it small. Each story should be able to be completed in 1 sprint.

    Remember the INVEST criteria:
      1. Independent: If stories rely on each other, consider combining them or re-sequencing
      2. Negotiable: If the story is overly detailed with technical specs or UI mockups, it’s not negotiable
      3. Valuable: Ask “Who benefits from this?” and “How does this help the user or business?”
      4. Estimable: You must be able to roughly estimate the effort or complexity of the story.
      5. Small: Small stories reduce risk, make progress visible, and give fast feedback.
      6. Testable: Vague stories like “Improve performance” or “Make it look better” are hard to test.

    Epic Title User Story Acceptance Criteria

  3. Now we're ready to prioritize! Create a backlog using the user stories above - breaking them down further as needed.

    The Backlog:

    Hints for Backlog Creation and Prioritization:
      1. Start with the MVP in mind.

      2. Your backlog should include more than just user stories.
        Make sure to include technical tasks, compliance tasks, accessibility reviews, etc. Just make sure it's trackable!

      3. Use relative sizing and don't aim for perfection.
        Estimation helps with sprint planning and capacity planning, not forecasting down to the hour.

    Sprint Review Best Practices:

    1. Focus on working software - demonstrate actual functionality, not mockups
    2. Be transparent about incomplete work - explain what's done, what's not, and why
